## 场景

```sql
EXPLAIN select * from employees where name > 'a';
```

![image-20211027034851438](/images/MySQL/image-20211027034851438.png)

```sql
EXPLAIN select * from employees where name > 'zzz';
```

![image-20211027034917587](/images/MySQL/image-20211027034917587.png)

对于上面这两种 name>'a' 和 name>'zzz' 的执行结果，mysql最终是否选择走索引或者一张表涉及多个索引，mysql最终如何选择索引，

涉及到mysql内部中有一个成本计算，该成本包括了回表的成本查询的成本

我们可以用**trace工具**来一查究竟，开启trace工具会影响mysql性能，所以只能临时分析sql使用，用完之后立即关闭

## trace工具

```sql
set session optimizer_trace="enabled=on",end_markers_in_json=on;  --开启trace
select * from employees where name > 'a' order by position;
SELECT * FROM information_schema.OPTIMIZER_TRACE;
```
