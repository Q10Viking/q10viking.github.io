# 安装Docker Desktop

win11系统开启适用于Linux的Windows子系统之后，就满足WSL 2 backend的环境条件，可以直接安装Docker Desktop

Install on Windows | Docker Documentationopen in new window

# 启动

启动docker-desktop软件后，子系统会启动类两个：docker-desktop和docker-desktop-data.这样在windows系统docker相关的命令才能使用,比如docker version命令。

PS C:\Users\11930\Desktop> wsl -l -v
  NAME                   STATE           VERSION
* docker-desktop         Running         2
  golang-centos          Stopped         2
  docker-desktop-data    Running         2

# 导出发行版

WSL 的基本命令 | Microsoft Learn： export-a-distribution-to-a-tar-fileopen in new window

# 停止
wsl -t docker-desktop-data
wsl -t docker-desktop
1
2
3
# 需要停止运行
PS C:\> wsl -l -v
  NAME                   STATE           VERSION
* Ubuntu                 Stopped         2
  docker-desktop-data    Stopped         2
  docker-desktop         Stopped         2
# 将分发版导出到 tar 文件。
PS C:\> wsl --export docker-desktop-data "E:\wsl\docker\docker-desktop-data.tar"
PS C:\> wsl --export docker-desktop "E:\wsl\docker\docker-desktop.tar"

# 导入发行版

WSL 的基本命令 | Microsoft Learn：import-a-new-distributionopen in new window

# 发行版名称  安装的位置  导入发行版的文件 使用的wsl版本2
PS C:\> wsl --import docker-desktop "E:\wsl\docker\docker-desktop" "E:\wsl\docker\docker-desktop.tar" --version 2
PS C:\> wsl --import docker-desktop-data "E:\wsl\docker\docker-desktop-data" "E:\wsl\docker\docker-desktop-data.tar" --version 2

PS C:\Users\11930\Desktop> wsl -l -v
  NAME                   STATE           VERSION

  docker-desktop-data    Running         2

修复方式，将docker-desktop设置为默认的发行版。WSL 设置默认的发行版 | Microsoft Learnopen in new window

PS C:\Users\11930\Desktop> wsl --set-default docker-desktop
PS C:\Users\11930\Desktop> wsl -l -v
  NAME                   STATE           VERSION
* docker-desktop         Running         2
  Ubuntu                 Stopped         2
  docker-desktop-data    Running         2

# Hytep-v
