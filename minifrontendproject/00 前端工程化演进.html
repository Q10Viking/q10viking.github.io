# 前端工程化演进思想

下面将用开发一个Todo List App来说明前端工程的演化，应用程序演化过程为：

  1. 最原始的html css javascript
  2. 然后到html scss typescript
  3. webpack工程化管理html scss typescript
  4. 引入现在流行的第三方前端框架vue.js
  5. 使用.vue单文件形式管理项目文件
  6. 使用现在vue推荐的CLI工具vite

# 阶段2 html scss typescript

Source Codeopen in new window

  1. 因为scss/sass的出现使得css在编码上更加的语义化和结构化，在前端开发样式这块引入了scss
  2. 通用的因为javascript是弱类型的语言，当项目变得很大，很多人参与的时候，为了以后的维护，引入了typescript来开发

但是浏览器只识别html css javascript。所以用scss和typescript编写的代码需要编译成css和javascript

将会用到下面这两个工具

  1. 使用tsc的工具初始化项目，具体参考[tsc管理ts文件](https://q10viking.github.io/typescript/04 tsc管理ts文件.html#单个文件),生成tsconfig.json文件
  2. 使用sass工具，具体参考Sassopen in new window

开发完应用程序后，分别用工具tsc编译typescript编写的.ts文件和sass工具编译scss编写的.scss文件，统一输出到dist文件下

# 一次性编译所有.ts文件
tsc
# Compile Sass to CSS 
sass style.scss dist/style.css

# 阶段3 使用webpack管理项目

Source Codeopen in new window

在阶段2中，开发完项目，都需要手动编译scss文件，ts文件。而webpack的出现,减轻了我们这些工作。它提供了插件sass-load,ts-loader等。通过一行命令就能编译项目中所有的代码。当然了webpack还提供了许多强大的功能。对于这篇文章我们关注的是ToDo List APP是如何逐渐工程化的。关于webpack更多的功能，查看webpack的官方文档open in new window

项目webpack5工程化具体步骤请参看这篇文章WebPack5工程化Typescript + Sassopen in new window

目录说明
scss存放scss的变量和mixin
ts存放公共的ts文件
srcapp.html,app.scss,app.ts文件就是应用

当运行配置配置的命令npm run build后就会生成相关的html,js,css文件,这些文件就是最终浏览器看到的。

# 配置webpack工程

  1. .vue格式的文件必须要被 @vue/compiler-sfcopen in new window 编译成标准的javascript和css,所以要使用单文件组件，需要安装这个
  2. vue-loader是在webpack工程下，加载.vue文件
# vue3中@vue/compiler-sfc替换了vue2的vue-template-compiler
npm install -D vue-loader  @vue/compiler-sfc
1
2

配置webpack.config.js Getting Started | Vue Loader (vuejs.org)open in new window

// webpack.config.js
const { VueLoaderPlugin } = require('vue-loader')

module.exports = {
    module: {
        rules: [
            // ... other rules
            {
                test: /\.vue$/,
                loader: 'vue-loader'
            },
            {
                test: /\.tsx?$/,
                //use: 'ts-loader', 
                loader: "ts-loader",
                options: {
                    appendTsSuffixTo: [/\.vue$/],     // 添加这行,支持在vue文件中的script lang="ts"
                },
                exclude: /node_modules/,
            }
        ]
    },
    resolve: {
        extensions: ['.tsx', '.ts', '.js'],
        alias: {
            vue: 'vue/dist/vue.esm-bundler.js',
            '@': path.resolve(__dirname,'src')   // 解决在ts文件中import App from '@/App.vue'
        }
    },
    plugins: [
        // make sure to include the plugin!
        new VueLoaderPlugin()
    ]
}

# Vite中使用Vue3+TS注意事项 ⭐

# defineComponent

TIP

To enable TypeScript supportopen in new window, wrap the component with defineComponent():

如果是这样定义

export default {
	data(){
        return {
            todoItems: []
        }
    }
}
1
2
3
4
5
6
7

会报错

error TS2339: Property 'todoItems' does not exist on type
1

使用defineComponent，才会进行类型推断

import { defineComponent } from 'vue'
export default defineComponent({
	data(){
        return {
            todoItems: []
        }
    }
})
