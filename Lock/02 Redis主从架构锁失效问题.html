# CAP角度分析redis和zk锁

redis主从架构满足的是AP架构。主从切换会导致锁的失效。

zk leader节点收到数据之后，会同步到slave子节点，有半数以上的节点同步成功，leader节点才响应客户端。如果主节点挂了，那么子节点成为主节点，数据仍然在。
