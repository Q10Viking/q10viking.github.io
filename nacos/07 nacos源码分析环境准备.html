# Nacos源码准备

  1. 在github上下载1.4.1的nacos版本https://github.com/alibaba/nacos/releases/tag/1.4.1

# 找启动类

  1. 在启动脚本中找到启动的类start.sh

    export SERVER="nacos-server"
JAVA_OPT="${JAVA_OPT} -jar ${BASE_DIR}/target/${SERVER}.jar"
nohup $JAVA ${JAVA_OPT} nacos.nacos >> ${BASE_DIR}/logs/start.out 2>&1 &
    1
    2
    3

  2. 找到运行的jar包 nacos-server.jar包,

    1. 进行解压

    2. 找到MANIFEST.MF文件

      Start-Class: com.alibaba.nacos.Nacos  // 该类为启动类
      1

# 在IDEA启动nacos

使用单机启动

-Dnacos.standalone=true
1

# 启动问题

# 找不到符号com.alibaba.nacos.consistency.entity

这个包目录是由protobuf在编译时自动生成，您可以通过mvn compile来自动生成他们。如果您使用的是IDEA，也可以使用IDEA的protobuf插件。

mvn compile
